# Brand And UX Gates

Use these gates before calling a screen grade A.

## Balencia Identity

- Dark-first, premium, warm, cinematic.
- Orange is primary action.
- Green is success, completion, or positive progress.
- Purple is SIA/AI.
- Domain colors should support domain identity, not take over global chrome.
- Sentence case for visible UI copy except approved eyebrow labels.

## SIA

- SIA should feel helpful, calm, and trustworthy.
- AI suggestions need clear action value.
- Avoid overusing purple, glow, or AI language where a simple task label is better.
- SIA should not interrupt sensitive flows unless the value is obvious.

## RPG And Motivation

- XP, streaks, missions, and achievements should motivate without making serious health or life goals feel childish.
- Celebration intensity should match the importance of the achievement.
- Mission language should be clear to new users.

## Forms And Permissions

- Ask for the minimum needed now.
- Explain sensitive asks before asking.
- Prefer progressive disclosure for profile fields, integrations, notifications, health data, photos, and social/accountability features.
- Offer escape hatches when a request is optional.

## Mobile Quality

- Primary actions must be reachable and visually dominant.
- Interactive targets should be at least 44x44pt.
- Text should be readable on a 375pt-width frame.
- Content should not collide with the dynamic island, tab bar, composer, or home indicator.
- Keyboard states should not trap or hide required actions.

## Control Purpose

- Every visible button, link, chip, row, icon button, field, tab, toggle, slider, and card CTA must have an intended user value.
- A visible control passes the A++ review only if it works in the prototype, is honestly disabled, is clearly decorative/static, or has an unambiguous documented Figma behavior.
- Controls that look actionable but do nothing create trust debt. Record them as findings with user impact and recommendation.
- Primary actions should answer: what happens, why now, and what the user gains.

## Figma Readiness

- The screen has a clear purpose and a clear next step within five seconds.
- The main state, empty/loading/error/disabled states, and key interaction states are either visible in evidence or explicitly documented for Figma.
- The screen can be handed to design without unresolved product, privacy, monetization, or navigation decisions.
- A++ requires composition, copy, hierarchy, state, and controls to be polished enough that Figma work is translation and refinement, not discovery.
