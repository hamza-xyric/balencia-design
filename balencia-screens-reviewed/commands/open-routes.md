# Open Routes

Use `batches/index.md` or the active batch file to identify the routes for the session.

Open one route at a time from:

```text
http://localhost:3000
```

Example:

```text
http://localhost:3000/auth/sign-up
```

Review the live route beside the matching spec in `../app_design 3/`.

After reviewing each route, write the complete screen note in the active batch file before opening the next route. Do not wait until the end of the session to compile notes from memory.

For the A++ re-review, also inventory visible controls before leaving the route. If a control looks tappable, record whether it works, is honestly disabled, is decorative/static, or needs documented Figma behavior.
