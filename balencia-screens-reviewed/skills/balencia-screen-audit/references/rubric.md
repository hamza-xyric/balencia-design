# Review Rubric

Score each dimension from 1 to 5.

## A++ Re-Review Grade Mapping

Use this mapping for the `a-plus-plus-review/` pass. The numeric scores still matter, but the letter grade is capped by unresolved findings and evidence quality.

| Grade | Required bar |
| --- | --- |
| A++ | All eight dimensions score `5`; fresh screenshot/browser evidence exists; no unresolved `critical`, `major`, or `minor` findings; only optional `nit` items remain; the screen is ready to translate into Figma without product or UX decisions. |
| A+ | No unresolved `critical` or `major` findings; one or more `minor` polish, copy, hierarchy, or documentation gaps remain before Figma. |
| A | The screen is coherent, safe, and directionally strong, but has meaningful polish, hierarchy, interaction, or Figma handoff issues. |
| B or below | The screen has unresolved critical task, trust, accessibility, mobile, brand, or product-sense risk. Record the blocker clearly instead of stretching the grade. |

Grade caps:

- Any unresolved `critical` finding caps the grade below `A`.
- Any unresolved `major` finding prevents `A+` or `A++`.
- Any unresolved `minor` finding prevents `A++`.
- Missing fresh route evidence prevents `A++`.
- A visible control with no clear purpose, no working prototype behavior, no honest disabled state, and no documented Figma behavior is at least a `major` finding.

## 5 - Excellent

The screen is clear, polished, emotionally appropriate, and aligned with Balencia. It has no obvious friction or trust gaps.

## 4 - Strong

The screen works well, with only small polish or clarity issues.

## 3 - Acceptable

The screen is usable, but has visible friction, unclear hierarchy, or missed brand/UX opportunities.

## 2 - Weak

The screen may confuse users, create avoidable friction, or feel below the intended quality bar.

## 1 - Blocking

The screen likely damages trust, conversion, comprehension, accessibility, or core task success.

## Dimensions

- Product sense: Does this screen belong here, and does it solve the right problem?
- User friction: Is the task as simple as it can reasonably be?
- Visual appeal: Does it feel premium and composed?
- Brand fit: Does it use Balencia's voice, color, typography, and SIA/RPG language correctly?
- Mobile ergonomics: Does it work as a touch-first mobile screen?
- Accessibility: Is it readable, navigable, and inclusive?
- Trust/privacy: Are sensitive asks, permissions, and AI claims handled responsibly?
- Industry best practice: Does it meet expectations set by strong consumer apps?
