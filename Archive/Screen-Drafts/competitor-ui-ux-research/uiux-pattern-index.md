# Cross-Competitor UI/UX Pattern Index

## Summary

Balencia's three primary references map cleanly to three design jobs:

- **Bevel:** Make complex personal data feel simple, calm, and actionable.
- **ChatGPT:** Make AI interaction feel immediate, flexible, and low-friction.
- **Habitica:** Make repeated effort feel rewarding, social, and visible over time.

Balencia should not merge the competitors literally. The best direction is to use ChatGPT as the SIA interaction model, Bevel as the domain-dashboard clarity model, and Habitica as the optional reward/social loop.

The YouTube evidence strengthens this direction: Bevel 3.0 pushes beyond static dashboards into an AI coach with memory/files, health records, suggested actions, and dashboard entry points; Habitica videos show that rewards and quests motivate best when they remain tied to real actions, not when configuration or punishment becomes the experience.

## Pattern Comparison

| Pattern | Bevel | ChatGPT | Habitica | Balencia implication |
|---|---|---|---|---|
| Primary surface | Health score dashboards and metric details | One universal chat timeline | Task lists and RPG progress | Today + SIA should be the main surfaces; domains are secondary depth. |
| Complexity handling | Interprets biometric data into scores and ranges | Lets the user ask naturally | Exposes task/game systems directly | SIA should hide complexity first, then reveal controls only when useful. |
| Visual hierarchy | One dominant score/chart per screen | Content-first message hierarchy | Dense list plus game status | Balencia dashboards should have one hero insight, then supporting actions. |
| AI interaction | Guidance cards and insights | Core interface and input model | Not central in captured UI | SIA needs ChatGPT-level input polish plus Bevel-like interpretation. |
| AI memory/control | Bevel Intelligence Files, health records, and settings appear as inspectable AI context | Memory/settings not captured in App Store evidence | Not central | SIA Memory should be visible, editable, and deleteable, with provenance for sensitive inputs. |
| Motivation | Readiness and non-judgmental guidance | Utility and immediate answers | XP, currency, quests, collections | Balencia should reward effort without making failure punitive. |
| Social accountability | Not visible in captured evidence | Not visible in captured evidence | Party quests and shared progress | Balencia social should move beyond leaderboards into shared goals. |
| Logging | Editable, task-specific forms | Conversational multimodal input | Task completion and forms | Allow both conversational logging and structured edit screens. |
| Empty/loading states | Calm, explanatory, time-aware | Minimal, composer-led | Progression/locked states | Design cold-start and data-sync states explicitly. |

## Balencia Borrow List

- Use Bevel's single-metric clarity for domain detail screens.
- Use ChatGPT's compact multimodal composer for SIA.
- Use Habitica's rewards and shared quests, but with mature visual treatment.
- Use Bevel-style pre-permission education before Apple Health, calendar, wearable, or notification asks.
- Use ChatGPT-style inline rich output inside SIA conversations.
- Use Habitica-style visible progression, but tie it to life domains and SIA plans.
- Use Bevel Intelligence-style files/memory for SIA context, with clear user control.
- Use Habitica-style custom rewards and skill/domain progression instead of fantasy classes as the default.

## Balencia Avoid List

- Avoid Bevel's narrow health-only framing.
- Avoid ChatGPT's lack of persistent progress/state.
- Avoid Habitica's overwhelming RPG-first surface for mainstream users.
- Avoid hard paywalling before users understand the coaching value.
- Avoid hiding edit controls after AI-generated logs or plans.
- Avoid making gamification setup, reward shops, or punishment loops more prominent than the user's real-world action.
- Avoid simple task buckets when the workflow needs domains, goals, projects, and recurring rituals.

## Adaptation Rules

- **For dashboards:** Start with Bevel. One score/insight first, clear next action second, history third.
- **For SIA:** Start with ChatGPT. One composer, multimodal input, rich inline answers, minimal chrome.
- **For gamification:** Start with Habitica. Completion should create XP/rewards/social momentum, but failure should not punish users by default.
- **For SIA memory:** Start with Bevel Intelligence Files. Give SIA durable context, but expose what it knows and let the user edit/delete it.
- **For monetization:** Show meaningful SIA value before subscription prompts. Bevel 3.0 critique makes clear that price can become the product story if value is not felt first.
- **For onboarding:** Combine Bevel's clarity with SIA's conversational flow. Keep progress visible and each step single-purpose.
- **For visual identity:** Keep Balencia's dark, warm, Sora-based system. Borrow interaction patterns, not competitor skins.
