# Close A++ Implementation Phase

Before marking an A++ implementation phase complete:

1. Confirm every addressed finding is listed in the active phase closeout.
2. Confirm every touched route/state has fresh browser evidence.
3. Confirm `npm run check` passed from `balencia-screens`.
4. Confirm `npm run build` passed when required or justified by shared/routing changes.
5. Confirm no A++ grade was changed in `a-plus-plus-review/index.md`.
6. Confirm unresolved product/scope/privacy questions are recorded in the active phase file and, if durable, in `findings/deferred-decisions.md`.
7. Confirm unrelated user changes were not reverted.
8. Update `a-plus-plus-implementation/index.md` only when the phase status truly changes.

Closeout must say exactly what was fixed, what evidence proves it, and what remains.
