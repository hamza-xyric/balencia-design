# Finding Taxonomy

## Severity

- `critical`: Blocks trust, conversion, comprehension, safety, or key task completion.
- `major`: Meaningfully hurts UX or brand quality and should be fixed before launch.
- `minor`: Noticeable issue that should be improved but does not block use.
- `nit`: Polish issue, wording tweak, or small consistency fix.

## Statuses

- `proposed`: recorded, waiting for decision.
- `accepted`: should become improvement work.
- `deferred`: valid, intentionally later.
- `rejected`: reviewed and decided against.
